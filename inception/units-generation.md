# Units Generation - Detailed Steps

## Overview
This stage decomposes the system into manageable units of work through two integrated parts:
- **Part 1 - Planning**: Create decomposition plan with questions, collect answers, analyze for ambiguities, get approval
- **Part 2 - Generation**: Execute approved plan to generate unit artifacts

## Prerequisites
- Workspace Detection must be complete
- Requirements Analysis recommended
- User Stories recommended
- Application Design stage REQUIRED

## Part 1: Planning
1. Create Unit of Work Plan
2. Include Mandatory Unit Artifacts in Plan
3. Generate Context-Appropriate Questions
4. Store UOW Plan
5. Request User Input
6. Collect Answers
7. Analyze Answers (MANDATORY)
8. MANDATORY Follow-up Questions
9. Request Approval
10. Log Approval
11. Update Progress

## Part 2: Generation
12. Load Unit of Work Plan
13. Execute Current Step
14. Update Progress
15. Continue or Complete
16. Present Completion Message
17. Wait for Explicit Approval
18. Record Approval Response
19. Update Progress

## Critical Rules
- NO HARDCODED LOGIC: Only execute what's written in the plan
- FOLLOW PLAN EXACTLY
- UPDATE CHECKBOXES immediately
- USE APPROVED APPROACH
- VERIFY COMPLETION
