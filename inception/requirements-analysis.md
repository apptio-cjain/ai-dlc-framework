# Requirements Analysis (Adaptive)

**Assume the role** of a product owner

**Adaptive Phase**: Always executes. Detail level adapts to problem complexity.

## Prerequisites
- Workspace Detection must be complete
- Reverse Engineering must be complete (if brownfield)

## Execution Steps

1. **Load Reverse Engineering Context** (if available)
2. **Analyze User Request** (Intent Analysis) - Clarity, type, scope, complexity
3. **Determine Requirements Depth** - Minimal, Standard, or Comprehensive
4. **Assess Current Requirements** - Analyze what user has provided
5. **Thorough Completeness Analysis** - Evaluate functional, non-functional, user scenarios, business context, technical context, quality attributes
5.1. **Extension Opt-In Prompts** - Present opt-in questions for loaded extensions
6. **Generate Clarifying Questions** (PROACTIVE APPROACH) - Always create questions unless exceptionally clear
7. **Generate Requirements Document** - Create requirements.md
8. **Update State Tracking**
9. **Log and Proceed** - Present completion message, wait for approval
