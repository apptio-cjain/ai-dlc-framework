# User Stories - Detailed Steps

## Purpose
**Convert requirements into user-centered stories with acceptance criteria**

## Prerequisites
- Workspace Detection must be complete
- Requirements Analysis recommended
- Workflow Planning must indicate User Stories stage should execute

## Intelligent Assessment Guidelines

### High Priority Execution (ALWAYS Execute)
- New user-facing features
- User experience changes
- Multi-persona systems
- Customer-facing APIs
- Complex business logic
- Cross-team projects

### Skip Only For Simple Cases
- Pure refactoring with zero user impact
- Isolated bug fixes
- Infrastructure only changes
- Developer tooling
- Documentation updates

## Part 1: Planning
1. Validate User Stories Need (MANDATORY)
2. Create Story Plan
3. Generate Context-Appropriate Questions
4. Include Mandatory Story Artifacts in Plan
5. Present Story Options
6. Store Story Plan
7. Request User Input
8. Collect Answers
9. Analyze Answers (MANDATORY)
10. MANDATORY Follow-up Questions
11. Wait for Explicit Approval of Plan

## Part 2: Generation
12. Load Story Generation Plan
13. Execute Current Step
14. Update Progress
15. Continue or Complete Generation
16. Present Completion Message
17. Wait for Explicit Approval of Generated Stories
18. Update Progress
